# ИИ-Агент СМК — актуальная документация проекта

Документ описывает текущее состояние проекта, его основные сценарии, новые функции, изменения в архитектуре и состав модулей.

**Дата актуализации:** 2026-04-16
**Источник правды по реализации:** [`main.py`](main.py), [`global_settings.json`](global_settings.json), [`local_settings.json`](local_settings.json)

## 1) Краткое описание проекта

**ИИ-Агент СМК** — Windows desktop-приложение на базе `CustomTkinter`, которое помогает работать с документами и артефактами системы менеджмента качества:

- выполнять консультации и аудит по документам СМК;
- искать релевантные фрагменты по базе знаний через RAG;
- читать Word, Excel, PDF, GraphML и legacy-документы;
- редактировать Word по индексам абзацев;
- искать и обновлять записи в Excel-журналах;
- генерировать новые документы Word и Excel;
- сохранять подтвержденные факты в долговременную память;
- восстанавливать старый контекст диалога из архивной памяти;
- готовить черновики email и встреч в Outlook;
- выполнять OCR-распознавание PDF-сканов через Vision-модели;
- сохранять и загружать сессии диалогов;
- работать в двух режимах доступа (Admin/Guest);
- индексировать сетевые папки и исключать файлы по ключевым словам;
- формировать кликабельные ссылки на файлы в ответах;
- искать информацию в интернете через Tavily API;
- искать определения и факты в Википедии;
- транскрибировать аудиофайлы (.mp3, .wav, .m4a, .ogg) через голосовые модели;
- навигировать по файловой базе через умный поиск файлов по категориям и ключевым словам;
- настраивать аудио-провайдеры (OpenRouter/Groq) и SOCKS5-прокси;
- индексировать документы из корпоративной XWiki через JSTree API;
- использовать голосовой ввод (dictation) с микрофона для ввода текста.

Ключевая идея проекта — **агентный цикл с Tool Calling**, где LLM не только отвечает текстом, но и сама выбирает нужные инструменты, выполняет действия и возвращает итог пользователю.

**Технологический стек:**
- GUI: customtkinter
- Векторная база: ChromaDB
- Работа с PDF: PyMuPDF (fitz)
- LLM API: OpenRouter (с поддержкой Vision- и Audio-моделей)
- XML/схемы: xml.etree.ElementTree
- Аудио: ffmpeg (ffprobe/ffmpeg для нарезки аудио)

## 2) Что изменилось в текущей версии

Ниже перечислены наиболее важные новые функции и архитектурные изменения относительно более ранней, упрощенной версии проекта.

### 2.1 Новые и расширенные функции

1. **Изоляция профиля пользователя**
   - Функция [`get_local_path()`](main.py:89) возвращает путь к изолированной папке профиля для конкретного экземпляра Агента.
   - Функция [`get_db_path()`](main.py:106) обеспечивает теневую репликацию: стягивает серверную БД на SSD пользователя для быстрой и безопасной работы.

2. **Vault-шифрование (AES-256)**
   - Функции [`get_vault_data()`](main.py:136) и [`save_vault_data()`](main.py:167) обеспечивают безопасное хранение чувствительных данных (API-ключей) с использованием шифрования Fernet.

3. **Динамические эмбеддинги**
   - Функция [`get_cloud_ef()`](main.py:190) реализует динамическую функцию эмбеддингов без глобальной инициализации, позволяя переключаться между моделями без перезапуска.

4. **Поиск в интернете через Tavily API**
   - Инструмент [`web_search_tavily()`](main.py:1555) выполняет поиск информации, новостей и актуальных требований во всем интернете. Возвращает до 5 источников с полным текстом и прямыми ссылками. Требует API-ключ в [`secrets.vault`](secrets.vault) (поле `tavily_key`).

5. **Поиск в Википедии**
   - Инструмент [`search_wikipedia_tool()`](main.py:1586) ищет термины, общие знания и определения в Википедии. Возвращает заголовок, краткое описание (до 2500 символов) и ссылку на статью.

6. **Интеллектуальный выбор источника поиска**
   - Системный промпт агента обновлен: теперь при отсутствии информации в локальной базе знаний агент предлагает пользователю выбрать источник (интернет или Википедия) и выполняет поиск только после явного согласия.

7. **Единая RAG-база по документам и памяти**
   - Индексация теперь строится вокруг коллекции `smk_docs` в ChromaDB, куда попадают документы из каталогов `SMK_Docs` и `Memory`.

8. **Инкрементальная переиндексация файлов**
   - Добавлен механизм сравнения временных меток файлов через [`sync_vector_db()`](main.py:1358) и файл состояния [`file_states.json`](file_states.json). Переиндексируются только новые, измененные или удаленные файлы.

9. **Поддержка legacy-форматов `.doc` и `.rtf`**
   - Функция [`convert_legacy_to_docx()`](main.py:1053) реализует COM-конвертацию Word-документов в `.docx` с сохранением результата в кэш `.cache/`, после чего текст читается обычным способом.

10. **Поддержка схем `.graphml`**
    - Функция [`extract_text_from_graphml()`](main.py:792) парсит схемы yEd через извлечение узлов, потоков, групп и маршрутизации в плоское текстовое описание для RAG.

11. **Умная работа с Excel**
    - Появились два специализированных инструмента:
    - [`smart_excel_search()`](main.py:1969) — поиск релевантных строк через LLM-разведку структуры листа и правил поиска;
    - [`smart_excel_edit()`](main.py:2077) — безопасное обновление или добавление строк в копию файла `*_Правки.xlsx`.

12. **Генерация документов с нуля**
    - Добавлены:
    - [`generate_document_from_scratch()`](main.py:2306) для Word-документов;
    - [`generate_excel_from_scratch()`](main.py:2412) для Excel-таблиц.

13. **Генерация документов по шаблону**
    - Инструмент [`generate_document_from_template()`](main.py:2212) создает новый документ на базе существующего шаблона с интеллектуальной заменой содержимого.

14. **Долговременная память и забывание фактов**
    - Появились функции:
    - [`memorize_important_fact()`](main.py:1600) — запись фактов в [`Memory/agent_memory.md`](Memory/agent_memory.md);
    - [`forget_fact()`](main.py:1613) — удаление фактов из памяти с последующей переиндексацией.

15. **Архив вытесненного контекста диалога**
    - Старые пары сообщений сохраняются в коллекцию `temp_chat_memory`, а затем могут быть извлечены через [`recall_past_conversation()`](main.py:1519).

16. **Фоновый "секретарь"**
    - Введен асинхронный анализ последних сообщений через [`run_background_secretary()`](main.py:3592), который автоматически сохраняет подтвержденные факты из диалога.

17. **Outlook-интеграция**
    - Добавлены инструменты:
    - [`draft_email_tool()`](main.py:2177) — черновик письма;
    - [`draft_meeting_tool()`](main.py:2193) — черновик встречи.

18. **Vision-роутинг для PDF и изображений**
    - Умный роутер [`extract_smart_vision_and_pdf()`](main.py:247) автоматически определяет, когда нужен OCR (мало текста, много изображений, схемы), и использует Vision-модель для распознавания. Включает mtime-кэширование.

19. **Настройки моделей через GUI**
    - Пользователь может менять основную модель, модель секретаря и модель эмбеддингов через окно настроек, открываемое из интерфейса.
    - Функция [`fetch_free_models()`](main.py:2847) загружает список бесплатных моделей из OpenRouter.

20. **Система сессий и история диалогов**
    - Реализовано автосохранение сессий чата в каталог [`Sessions/`](Sessions/) в формате JSON. Каждая сессия содержит уникальный ID, заголовок (генерируется автоматически через LLM), временную метку, историю сообщений и отображаемый текст.
    - Функция [`generate_session_title_background()`](main.py:3436) генерирует заголовки сессий в фоновом режиме.
    - Добавлена кнопка «История диалогов» в боковой панели для просмотра, загрузки и удаления старых сессий через [`open_history_window()`](main.py:3511).

21. **Система ролей (Admin/Guest)**
    - Введена двухуровневая система доступа:
    - **Admin** — полный доступ: сохранение сессий, запись фактов в память, архивация чата в векторную базу;
    - **Guest** — ограниченный доступ: данные не сохраняются на диск, факты не записываются в память.
    - Аутентификация выполняется по паролю через [`prompt_auth()`](main.py:2867), заданному в [`secrets.vault`](secrets.vault) (поле `admin_password`).

22. **Расширенная конфигурация индексации**
    - Добавлены новые параметры в [`global_settings.json`](global_settings.json):
    - `indexed_folders` — массив путей к папкам для индексации (поддержка сетевых путей, например `Z:/...`);
    - `exclude_keywords` — массив ключевых слов для исключения файлов из индексации (например, «архив», «not_index», «черновик»);
    - `audio_provider` / `audio_model` — настройки аудио-транскрибации;
    - `audio_chunk_mins` / `audio_overlap_secs` — параметры нарезки аудио.

23. **Кликабельные ссылки на файлы в ответах агента**
    - Агент теперь автоматически формирует ссылки на упомянутые документы в формате `[Из файла: Имя_файла.ext]`. При клике на ссылку файл открывается в системном приложении через [`on_link_click()`](main.py:3380).

24. **Улучшенный GUI**
    - Многострочное текстовое поле ввода (3-4 строки) с поддержкой Shift+Enter для переноса строки;
    - Стили бейджей (плашек) для сообщений пользователя и агента;
    - Прогресс-бар синхронизации базы знаний через [`update_progress_ui()`](main.py:3195);

25. **Генерация диаграмм Mermaid и yEd**
    - Добавлены инструменты [`generate_mermaid_diagram()`](main.py:1628) и [`generate_yed_diagram()`](main.py:1790) для создания визуальных схем по описанию. Позволяют генерировать блок-схемы, процессные диаграммы и графики в форматах Mermaid HTML и yEd GraphML.

27. **Единый локатор файлов**
    - Функция [`find_target_file()`](main.py:1080) выполняет интеллектуальный поиск файлов с учетом черного списка ключевых слов и настроек индексации, обеспечивая точное нахождение целевых документов.

28. **Сканирование папок для документов**
    - Функция [`scan_folders_for_docs()`](main.py:1219) рекурсивно собирает список файлов из указанных папок, исключая файлы по ключевым словам, и возвращает готовый список для индексации.

    - Функции [`get_file_states()`](main.py:1265) и [`save_file_states()`](main.py:1273) обеспечивают механизм инкрементальной индексации, отслеживая изменения файлов через временные метки и сохраняя состояние в [`file_states.json`](file_states.json).

### 2.2 Архитектурные изменения

1. **Система настроек с вкладками**
   - Окно настроек [`App.open_settings()`](main.py:3665) теперь организовано по вкладкам:
   - «Модели» — выбор основной, Vision, секретаря и эмбеддинг-моделей;
   - «Исключения» — управление ключевыми словами для исключения из индексации;
   - «Папки» — интерактивное управление папками для индексации (добавление/удаление);
   - «Безопасность» (только Admin) — API-ключи и пароль администратора;
   - «Аудио и Сеть» (только Admin) — аудио-провайдер, модель, параметры нарезки, прокси;
   - «О программе» — информация о версии и разработчике.

2. **Разделение настроек на локальные и глобальные**
   - [`local_settings.json`](local_settings.json) — хранит пользовательские настройки (модели, история моделей, прокси);
   - [`global_settings.json`](global_settings.json) — хранит общие настройки (Vision, секретарь, эмбеддинги, аудио, папки, исключения);
   - [`secrets.vault`](secrets.vault) — зашифрованное хранилище API-ключей (OpenRouter, Groq, Tavily, admin_password).

3. **Переход к более явной агентной архитектуре**
   - Центральная логика сосредоточена в агентном цикле [`App.agent_loop()`](main.py:4395), где системная инструкция задает строгий порядок вызова инструментов.

4. **Разделение памяти на три уровня**
   - оперативная история чата — [`chat_history.json`](chat_history.json);
   - долговременные факты — [`Memory/agent_memory.md`](Memory/agent_memory.md);
   - архив вытесненного диалога — коллекция `temp_chat_memory`.

5. **Появление кэш-слоя для промежуточных преобразований**
   - Каталог `.cache/` используется для:
   - конвертированных legacy-документов (`{name}_converted.docx`);
   - markdown-копий распознанных схем GraphML (`{name}_schema.md`);
   - результатов Vision/OCR-распознавания PDF-сканов (`{filename}_vision.md`);
   - расшифровок аудиофайлов (`{name}_audio.md`).

6. **Инкрементальная индексация вместо полного сканирования "вслепую"**
   - Архитектура индексации учитывает измененные и удаленные файлы, снижая стоимость и время синхронизации.

7. **Многопоточность для тяжелых операций**
   - Все тяжелые операции (парсинг, API-вызовы, эмбеддинг, транскрибация) выполняются в отдельных потоках через `threading.Thread`:
   - синхронизация БД при старте приложения;
   - переиндексация после генерации документов;
   - агентный цикл обработки сообщений;
   - фоновый анализ диалога секретарем;
   - транскрибация аудиофайлов.
   - Обновление GUI выполняется через `self.after(0, lambda: ...)`.

8. **Система сессий с автосохранением**
   - Реализован четырехуровневый подход к хранению данных:
   - оперативная история чата — [`chat_history.json`](chat_history.json) (глобальная);
   - сессии диалогов — [`Sessions/`](Sessions/) (JSON-файлы с уникальным ID);
   - долговременные факты — [`Memory/agent_memory.md`](Memory/agent_memory.md);
   - архив вытесненного диалога — коллекция `temp_chat_memory` (только для Admin).

9. **Разделение доступа на основе ролей**
   - Введена система аутентификации с двумя ролями:
   - **Admin** — может сохранять сессии, записывать факты в память, архивировать чат;
   - **Guest** — работает в режиме «только чтение», данные не сохраняются на диск.
   - Пароль задается в [`secrets.vault`](secrets.vault), аутентификация через GUI.

10. **Расширенная индексация с поддержкой сетевых путей и исключений**
    - Функция [`sync_vector_db()`](main.py:1358) теперь:
    - сканирует не только локальные папки, но и сетевые пути (например, `Z:/...`);
    - исключает файлы по ключевым словам из `exclude_keywords` (например, «архив», «not_index»);
    - использует расширенный список папок из `indexed_folders` в [`global_settings.json`](global_settings.json).

11. **Добавление слоя веб-поиска**
    - В архитектуру добавлен новый слой инструментов для поиска информации за пределами локальной базы знаний:
    - [`web_search_tavily()`](main.py:1555) — поиск в интернете через Tavily API;
    - [`search_wikipedia_tool()`](main.py:1586) — поиск в Википедии.
    - Инструменты интегрированы в агентный цикл с интеллектуальным выбором источника поиска.

12. **Изоляция профиля и теневые реплики**
    - Функции [`get_local_path()`](main.py:89) и [`get_db_path()`](main.py:106) обеспечивают работу нескольких экземпляров Агента на одном ПК с изолированными данными.
    - Vault-шифрование через Fernet защищает чувствительные данные.

13. **Синхронизация с XWiki**
    - Функция [`sync_xwiki()`](main.py:371) реализует иерархический обход дерева документов XWiki через JSTree API.
    - Поддерживается индексация документов из корпоративной XWiki.
    - Интеграция настраивается через вкладку «XWiki» в окне настроек.

### 3.1 Основные слои

- чат с поддержкой markdown-разметки и кликабельных ссылок ([`apply_markdown()`](main.py:3300), [`on_link_click()`](main.py:3380));
- многострочное текстовое поле ввода (3-4 строки) с поддержкой Shift+Enter;

- кнопку очистки чата;
- кнопку «История диалогов» для просмотра, загрузки и удаления старых сессий ([`open_history_window()`](main.py:3511));

- рендер Unicode-таблиц ([`generate_unicode_table()`](main.py:3212));
- стили бейджей (плашек) для сообщений пользователя и агента;
- диалог сохранения при генерации документов ([`ask_save_path_sync()`](main.py:3169)).

#### Агентный слой

Агентный цикл реализован в [`App.agent_loop()`](main.py:4395). Он:

- формирует системный промпт со строгим алгоритмом действий (9 шагов + шаги 1.1–1.4 для веб-поиска, аудио и навигации);
- передает модели схему доступных инструментов ([`get_tools_schema()`](main.py:4007));
- обрабатывает `tool_calls` ([`execute_tool()`](main.py:4340));
- ограничивает цикл 10 шагами;
- архивирует старые сообщения после переполнения окна истории (только для Admin);
- генерирует кликабельные ссылки на файлы в формате `[Из файла: Имя_файла.ext]`;
- использует потоковую (stream) генерацию ответов для плавного отображения текста.

#### Слой голосового ввода

Реализован классом [`AudioRecorder`](main.py:2616):

- [`get_microphones()`](main.py:2622) — возвращает список доступных микрофонов;
- [`start_recording()`](main.py:2640) — запуск записи с указанного устройства;
- [`stop_recording()`](main.py:2656) — остановка записи и сохранение в WAV-файл;
- [`_on_record_start()`](main.py:2936) — обработчик начала записи;
- [`_on_record_stop()`](main.py:2956) — обработчик остановки и запуска транскрибации;
- [`_insert_transcript()`](main.py:3074) — вставка распознанного текста в поле ввода;
- [`apply_audio_hotkey()`](main.py:2912) — привязка динамических горячих клавиш.

#### Слой XWiki-интеграции

Функции для работы с XWiki:

- [`sync_xwiki()`](main.py:371) — иерархический обход дерева документов XWiki через JSTree API;
- [`update_xwiki_progress()`](main.py:734) — обновление прогресса синхронизации в UI;
- [`format_xwiki_url_for_ui()`](main.py:2682) — форматирование XWiki URL для отображения.

#### Слой инструментов

Инструменты объявляются в [`App.get_tools_schema()`](main.py:4007) и маршрутизируются через [`App.execute_tool()`](main.py:4340).

#### Слой веб-поиска

Добавлены инструменты для поиска информации за пределами локальной базы знаний:
- [`web_search_tavily()`](main.py:1555) — поиск в интернете через Tavily API;
- [`search_wikipedia_tool()`](main.py:1586) — поиск в Википедии.

#### Слой знаний и индексации

Индексация строится вокруг функции [`sync_vector_db()`](main.py:1358), которая:

- сканирует папки из `indexed_folders` в [`global_settings.json`](global_settings.json) (включая сетевые пути);
- исключает файлы по ключевым словам из `exclude_keywords` (например, «архив», «not_index»);
- пропускает каталог `.cache`;
- читает поддерживаемые форматы;
- разбивает текст на чанки через [`chunk_text()`](main.py:1210);
- сохраняет чанки в коллекцию `smk_docs`;
- удаляет записи для исчезнувших файлов.

#### Слой памяти

Память состоит из нескольких механизмов:

- история текущего чата сохраняется через [`App.save_history()`](main.py:3422);
- сессии диалогов автосохраняются в [`Sessions/`](Sessions/) через [`App.save_current_session()`](main.py:3460);
- заголовки сессий генерируются автоматически через [`App.generate_session_title_background()`](main.py:3436);
- подтвержденные факты пишутся в [`Memory/agent_memory.md`](Memory/agent_memory.md) (только для Admin);
- старые пары сообщений архивируются в `temp_chat_memory` в блоке переполнения истории внутри [`App.agent_loop()`](main.py:4395) (только для Admin).

#### Слой изоляции профиля

Слой обеспечивает:
- [`get_local_path()`](main.py:89) — изоляция данных каждого экземпляра Агента;
- [`get_db_path()`](main.py:106) — теневая репликация БД на локальный SSD;
- [`get_vault_data()`](main.py:136) / [`save_vault_data()`](main.py:167) — шифрование чувствительных данных через Fernet.

## 5) Поддерживаемые форматы и источники данных

Функция [`read_local_file()`](main.py:735) поддерживает:

- `.txt`
- `.md`
- `.docx`
- `.doc`
- `.rtf`
- `.pdf`
- `.xlsx`
- `.xls`
- `.graphml`
- `.html`
- `.mp3`
- `.wav`
- `.m4a`
- `.ogg`

Дополнительно:

- если передать путь к папке, инструмент возвращает список доступных файлов, а не пытается читать директорию как документ;
- поиск файла выполняется в текущем пути, а затем рекурсивно внутри `SMK_Docs` и `Memory`;
- для `.doc/.rtf` используется промежуточная COM-конвертация;
- для `.graphml` создается текстовое описание, пригодное для RAG;
- для аудиофайлов проверяется наличие кэша расшифровки, иначе предлагается запустить транскрибацию.

## 7) Установка и запуск

### 7.1 Предварительные требования

- Windows 10/11;
- Python 3.10+;
- доступ в интернет для OpenRouter API;
- установленный Microsoft Word/Outlook для COM-функций;
- `ffmpeg` и `ffprobe` в PATH системы (для транскрибации аудио);

### 7.2 Установка

```cmd
python -m venv .venv
.venv\Scripts\activate
pip install --upgrade pip
pip install chromadb python-docx python-dotenv openai customtkinter keyboard openpyxl pywin32 PyMuPDF requests wikipedia cryptography
```

В [`.env`](.env) должен быть минимум один ключ:

```env
OPENROUTER_API_KEY=ваш_ключ
```

Вариант 1:

```cmd
start.bat
```

Вариант 2:

```bash
python main.py
```

- создает каталоги `Memory` и `SMK_Docs`, если их нет;
- запускает фоновую синхронизацию базы знаний;
- загружает список бесплатных моделей из OpenRouter;
- отображает статус готовности базы в интерфейсе.

## 8) Конфигурация

### 8.1 Файл настроек

Конфигурация хранится в [`global_settings.json`](global_settings.json). Ключевые параметры:

- `vision_model` — модель для Vision OCR (используется в экспериментальном контуре);
- `secretary_model` — модель фонового секретаря;
- `embedding_model` — модель облачных эмбеддингов;
- `audio_provider` — провайдер аудио-транскрибации (OpenRouter/Groq);
- `audio_model` — модель для аудио-транскрибации;
- `audio_chunk_mins` — длительность одного аудио-фрагмента (мин);
- `audio_overlap_secs` — перекрытие между аудио-фрагментами (сек);
- `indexed_folders` — массив путей к папкам для индексации (поддержка сетевых путей, например `Z:/...`);
- `exclude_keywords` — массив ключевых слов для исключения файлов из индексации (например, «архив», «not_index», «черновик»);
- `default_excel_file` — Excel-файл по умолчанию;
- `excel_status_col` — колонка статуса в Excel;
- `excel_open_val` / `excel_closed_val` — допустимые значения статусов;
- `chroma_batch_size` — размер батча при индексации.

#### Локальные настройки (local_settings.json)

| Параметр | Описание |
|----------|----------|
| `guest_model` | Модель для режима Guest (бесплатная) |
| `admin_model` | Модель для режима Admin |
| `model_history` | История использованных моделей (Топ-10) |
| `use_proxy` | Флаг использования SOCKS5-прокси |
| `proxy_host` | Хост прокси (по умолчанию 127.0.0.1) |
| `proxy_port` | Порт прокси (по умолчанию 2080) |

#### Зашифрованный Vault (secrets.vault)

| Параметр | Описание |
|----------|----------|
| `openrouter_key` | API-ключ OpenRouter |
| `groq_key` | API-ключ Groq (для аудио) |
| `tavily_key` | API-ключ Tavily (для поиска в интернете) |
| `admin_password` | Пароль администратора |

### 8.2 GUI-настройки

Через окно настроек [`App.open_settings()`](main.py:2824) с вкладками можно настроить:

#### Вкладка «Модели»
- модель агента (Admin/Guest);
- модель для Vision OCR;
- модель фонового секретаря;
- модель эмбеддингов (требует перезапуска).

#### Вкладка «Исключения»
- ключевые слова для исключения файлов/папок из индексации.

#### Вкладка «Папки»
- добавление и удаление папок для индексации (только Admin).

#### Вкладка «Безопасность» (только Admin)
- OpenRouter API Key;
- Groq API Key;
- Tavily API Key;
- Пароль администратора.

#### Вкладка «Аудио и Сеть» (только Admin)
- аудио-провайдер (OpenRouter/Groq);
- аудио-модель (с автозагрузкой доступных моделей);
- длительность куска (мин) и перекрытие (сек);
- использование SOCKS5-прокси (host/port).

#### Вкладка «О программе»
- информация о версии, разработчике и контактах.

#### Вкладка «XWiki» (только Admin)
- добавление и удаление URL-ссылок на разделы XWiki;
- управление синхронизацией документов из корпоративной XWiki.

Изменение `embedding_model` требует перезапуска приложения, поскольку эмбеддинг-функция инициализируется при старте.
