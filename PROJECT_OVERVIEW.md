## 1) Краткое описание проекта

**ИИ-Агент СМК** — Windows desktop-приложение на базе `CustomTkinter`, которое помогает работать с документами и артефактами системы менеджмента качества:

- выполнять консультации и аудит по документам СМК;
- искать релевантные фрагменты по базе знаний через RAG;
- читать Word, Excel, PDF, GraphML и legacy-документы;
- редактировать Word по индексам абзацев;
- искать и обновлять записи в Excel-журналах;
- генерировать новые документы Word и Excel;
- сохранять подтвержденные факты в долговременную память;
- восстанавливать старый контекст диалога из архивной памяти;
- готовить черновики email и встреч в Outlook;
- выполнять OCR-распознавание PDF-сканов через Vision-модели;
- сохранять и загружать сессии диалогов;
- работать в двух режимах доступа (Admin/Guest);
- индексировать сетевые папки и исключать файлы по ключевым словам;
- формировать кликабельные ссылки на файлы в ответах;
- искать информацию в интернете через Tavily API;
- искать определения и факты в Википедии.

Ключевая идея проекта — **агентный цикл с Tool Calling**, где LLM не только отвечает текстом, но и сама выбирает нужные инструменты, выполняет действия и возвращает итог пользователю.

**Технологический стек:**
- GUI: customtkinter
- Векторная база: ChromaDB
- Работа с PDF: PyMuPDF (fitz)
- LLM API: OpenRouter (с поддержкой Vision-моделей)
- XML/схемы: xml.etree.ElementTree

### 2.1 Новые и расширенные функции

1. **Изоляция профиля пользователя**
   - Функция [`get_local_path()`](main.py:75) возвращает путь к изолированной папке профиля для конкретного экземпляра Агента.
   - Функция [`get_db_path()`](main.py:92) обеспечивает теневую репликацию: стягивает серверную БД на SSD пользователя для быстрой и безопасной работы.

2. **Vault-шифрование (AES-256)**
   - Функции [`get_vault_data()`](main.py:122) и [`save_vault_data()`](main.py:147) обеспечивают б��зопасное хранение чувствительных данных (API-ключей) с использованием шифрования Fernet.

3. **Динамические эмбеддинги**
   - Функция [`get_cloud_ef()`](main.py:167) реализует динамическую функцию эмбеддингов без глобальной инициализации, позволяя переключаться между моделями без перезапуска.

4. **Поиск в интернете через Tavily API**
   - Инструмент [`web_search_tavily()`](main.py:862) выполняет поиск информации, новостей и актуальных требований во всем интернете. Возвращает до 5 источников с полным текстом и прямыми ссылками. Требует API-ключ в [`global_settings.json`](global_settings.json) (поле `tavily_api_key`).

5. **Поиск в Википедии**
   - Инструмент [`search_wikipedia_tool()`](main.py:893) ищет термины, общие знания и определения в Википедии. Возвращает заголовок, краткое описание (до 2500 символов) и ссылку на статью.

6. **Интеллектуальный выбор источника поиска**
   - Системный промпт агента обновлен: теперь при отсутствии информации в локальной базе знаний агент предлагает пользователю выбрать источник (интернет или Википедия) и выполняет поиск только после явного согласия.

7. **Единая RAG-база по документам и памяти**
   - Индексация теперь строится вокруг коллекции `smk_docs` в ChromaDB, куда попадают документы из каталогов `SMK_Docs` и `Memory`.

8. **Инкрементальная переиндексация файлов**
   - Добавлен механизм сравнения временных меток файлов через [`sync_vector_db()`](main.py:715) и файл состояния [`file_states.json`](file_states.json). Переиндексируются только новые, измененные или удаленные файлы.

9. **Поддержка legacy-форматов `.doc` и `.rtf`**
   - Функция [`convert_legacy_to_docx()`](main.py:536) реализует COM-конвертацию Word-документов в `.docx` с сохранением результата в кэш `SMK_Docs/.cache/`, после чего текст читается обычным способом.

10. **Поддержка схем `.graphml`**
    - Функция [`extract_text_from_graphml()`](main.py:399) парсит схемы yEd через извлечение узлов, потоков, групп и маршрутизации в плоское текстовое описание для RAG.

11. **Умная работа с Excel**
    - Появились два специализированных инструмента:

    - [`smart_excel_edit()`](main.py:1384) — безопасное обновление или добавление строк в копию файла `*_Правки.xlsx`.

12. **Генерация документов с нуля**

19. **Настройки моделей через GUI**
    - Пользователь может менять основную модель, модель секретаря и модель эмбеддингов через окно настроек, открываемое из интерфейса.

### 2.2 Архитектурные изменения

1. **Переход к более явной агентной архитектуре**
   - Центральная логика сосредоточена в агентном цикле [`App.agent_loop()`](main.py:3083), где системная инструкция задает строгий порядок вызова инструментов.

2. **Разделение памяти на три уровня**
   - оперативная история чата — [`chat_history.json`](chat_history.json);
   - долговременные факты — [`Memory/agent_memory.md`](Memory/agent_memory.md);
   - архив вытесненного диалога — коллекция `temp_chat_memory`.

3. **Появление кэш-слоя для промежуточных преобразований**
   - Каталог `SMK_Docs/.cache/` используется для:
   - конвертированных legacy-документов (`{name}_converted.docx`);
   - markdown-копий распознанных схем GraphML (`{name}_schema.md`);
   - результатов Vision/OCR-распознавания PDF-сканов (`{filename}_vision.md`).

4. **Инкрементальная индексация вместо полного сканирования "вслепую"**
   - Архитектура индексации учитывает измененные и удаленные файлы, снижая стоимость и время синхронизации.

5. **Многопоточность для тяжелых операций**
   - Все тяжелые операции (парсинг, API-вызовы, эмбеддинг) выполняются в отдельных потоках через `threading.Thread`:
   - синхронизация БД при старте приложения;
   - переиндексация после генерации документов;
   - агентный цикл обработки сообщений;
   - фоновый анализ диалога секретарем.
   - Обновление GUI выполняется через `self.after(0, lambda: ...)`.

6. **Система сессий с автосохранением**
   - Реализован четырехуровневый подход к хранению данных:
   - оперативная история чата — [`chat_history.json`](chat_history.json) (глобальная);
   - сессии диалогов — [`Sessions/`](Sessions/) (JSON-файлы с уникальным ID);
   - долговременные факты — [`Memory/agent_memory.md`](Memory/agent_memory.md);
   - архив вытесненного диалога — коллекция `temp_chat_memory` (только для Admin).

7. **Разделение доступа на основе ролей**
   - Введена система аутентификации с двумя ролями:
   - **Admin** — может сохранять сессии, записывать факты в память, архивировать чат;
   - **Guest** — работает в режиме «только чтение», данные не сохраняются на диск.
   - Пароль задается в [`global_settings.json`](global_settings.json), аутентификация через GUI.

8. **Расширенная индексация с поддержкой сетевых путей и исключений**
   - Функция [`sync_vector_db()`](main.py:715) теперь:
   - сканирует не только локальные папки, но и сетевые пути (например, `Z:/...`);
   - исключает файлы по ключевым словам из `exclude_keywords` (например, «архив», «not_index»);
   - использует расширенный список папок из `indexed_folders` в [`global_settings.json`](global_settings.json).

9. **Добавление слоя веб-поиска**
   - В архитектуру добавлен новый слой инструментов для поиска информации за пределами локальной базы знаний:
   - [`web_search_tavily()`](main.py:862) — поиск в интернете через Tavily API;
   - [`search_wikipedia_tool()`](main.py:893) — поиск в Википедии.
   - Инструменты интегрированы в агентный цикл с интеллектуальным выбором источника поиска.

10. **Изоляция профиля и теневые реплики**
    - Функции [`get_local_path()`](main.py:75) и [`get_db_path()`](main.py:92) обеспечивают работу нескольких экземпляров Агента на одном ПК с изолированными данными.
    - Vault-шифрование черезFernet защищает чувствительные данные.

### 3.1 Основные слои

#### GUI-слой

Интерфейс приложения реализован в классе [`App`](main.py:1911) и включает:

- чат с поддержкой markdown-разметки и кликабельных ссылок ([`apply_markdown()`](main.py:2215), [`on_link_click()`](main.py:2295));
- многострочное текстовое поле ввода (3-4 строки) с поддержкой Shift+Enter;
- кнопку синхронизации базы с прогресс-баром ([`update_progress_ui()`](main.py:2110));
- кнопку очистки чата;
- кнопку «История диалогов» для просмотра и загрузки старых сессий ([`open_history_window()`](main.py:2413));
- кнопку «Войти как Админ» с аутентификацией по паролю ([`prompt_auth()`](main.py:2055));
- окно настройки моделей ([`open_settings()`](main.py:2566));
- рендер Unicode-таблиц ([`generate_unicode_table()`](main.py:2127));
- стили бейджей (плашек) для сообщений пользователя и агента;
- диалог сохранения при синхронизации ([`ask_save_path_sync()`](main.py:2084)).

#### Агентный слой

Агентный цикл реализован в [`App.agent_loop()`](main.py:3083). Он:

- формирует системный промпт со строгим алгоритмом действий (8 шагов + шаги 1.1 и 1.2 для веб-поиска);
- передает модели схему доступных инструментов ([`get_tools_schema()`](main.py:2740));
- обрабатывает `tool_calls` ([`execute_tool()`](main.py:3038));
- ограничивает цикл 10 шагами;
- архивирует старые сообщения после переполнения окна истории (только для Admin);
- генерирует кликабельные ссылки на файлы в формате `[Из файла: Имя_файла.ext]`.

#### Слой инструментов

Инструменты объявляются в [`App.get_tools_schema()`](main.py:2740) и маршрутизируются через [`App.execute_tool()`](main.py:3038).

#### Слой веб-поиска

Добавлены инструменты для поиска информации за пределами локальной базы знаний:
- [`web_search_tavily()`](main.py:862) — поиск в интернете через Tavily API;
- [`search_wikipedia_tool()`](main.py:893) — поиск в Википедии.

#### Слой знаний и индексации

Индексация строится вокруг функции [`sync_vector_db()`](main.py:715), которая:

- сканирует папки из `indexed_folders` в [`global_settings.json`](global_settings.json) (включая сетевые пути);
- исключает файлы по ключевым словам из `exclude_keywords` (например, «архив», «not_index»);
- пропускает каталог `.cache`;
- читает поддерживаемые форматы;
- разбивает текст на чанки через [`chunk_text()`](main.py:654);
- сохраняет чанки в коллекцию `smk_docs`;
- удаляет записи для исчезнувших файлов.

#### Слой памяти

Память состоит из нескольких механизмов:

- история текущего чата сохраняется через [`App.save_history()`](main.py:2334);
- сессии диалогов автосохраняются в [`Sessions/`](Sessions/) через [`App.save_current_session()`](main.py:2364);
- заголовки сессий генерируются автоматически через [`App.generate_session_title_background()`](main.py:2340);
- подтвержденные факты пишутся в [`Memory/agent_memory.md`](Memory/agent_memory.md) (только для Admin);
- старые пары сообщений архивируются в `temp_chat_memory` в блоке переполнения истории внутри [`App.agent_loop()`](main.py:3083) (только для Admin).

#### Слой изоляции профиля

Новый слой обеспечивает:
- [`get_local_path()`](main.py:75) — изоляция данных каждого экземпляра Агента;
- [`get_db_path()`](main.py:92) — теневая репликация БД на локальный SSD;
- [`get_vault_data()`](main.py:122) / [`save_vault_data()`](main.py:147) — шифрование чувствительных данных через Fernet.

## 8) Конфигурация

### 8.1 Файл настроек

Конфигурация хранится в [`global_settings.json`](global_settings.json). Ключевые параметры:

- `model` — основная модель агента;
- `secretary_model` — модель фонового секретаря;
- `embedding_model` — модель облачных эмбеддингов;
- `vision_model` — модель для Vision OCR (используется в экспериментальном контуре);
- `roles` — система ролей агента (Консультант, Аудитор, Методолог);
- `default_excel_file` — Excel-файл по умолчанию;
- `excel_status_col` — колонка статуса в Excel;
- `excel_open_val` / `excel_closed_val` — допустимые значения статусов;
- `chroma_batch_size` — размер батча при индексации;
- `indexed_folders` — массив путей к папкам для индексации (поддержка сетевых путей, например `Z:/...`);
- `exclude_keywords` — массив ключевых слов для исключения файлов из индексации (например, «архив», «not_index», «черновик»);
- `model_history` — история использованных моделей для быстрого переключения;
- `admin_password` — пароль администратора для аутентификации;
- `tavily_api_key` — API-ключ для поиска в интернете через Tavily.

#### Система ролей

В [`global_settings.json`](global_settings.json) доступны три предопределенные роли:

| Роль | Назначение |
|------|------------|
| **Консультант** | Умный помощник по СМК. Отвечает на вопросы по документам четко и по делу. |
| **Аудитор** | Строгий ИИ-аудитор СМК. Ищет несоответствия стандартам. Объективен, пишет кратко и только по делу. |
| **Методолог** | Специалист по написанию регламентов. Создает структурированные документы с шапкой и подписями. |

Переключение между ролями осуществляется через настройки в интерфейсе.

Актуальные значения задаются в [`global_settings.json`](global_settings.json) и загружаются при старте приложения.

### 8.2 GUI-настройки

Через окно, открываемое из [`App.open_settings()`](main.py:2566), можно изменить:

- модель агента;
- модель секретаря;
- модель эмбеддингов.

Изменение `embedding_model` требует перезапуска приложения, поскольку эмбеддинг-функция инициализируется при старте.
